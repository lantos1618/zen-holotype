# Zen Goals

Compact roadmap from the current codebase. Each goal names the gap and the check that proves it is done.

1. **Namespaces**
   - Status: struct methods can have their own type params; `Vec<T>` owns `push/grow/free`, `Map<T>` owns `put/try_put/free`; namespace-bound collection construction now has natural public `vec.of`/`vec.try_of` and `maps.of`/`maps.try_of`; runtime uses `rt.sync`/`rt.async` plus receiver `free`, with the old `sync_arena`/`async_arena` aliases removed; cown's public surface is now `cown.buf`/`cown.file`/`cown.file_in` instead of exported `buf_alloc`/`file_open`; IO now exposes `file.contents`/`file.contents_in`/`file.save`/`file.shell` plus `file.out`/`file.out_bytes`, with old public `read_file`/`read_file_in`/`write_file`/`run_cmd`/`write_stdout`/`write_stdout_bytes` spellings removed; numeric text exposes allocator-first `num.integer(a, ...)`/`num.float(a, ...)` with no default-heap wrappers and with the old public `int_string`/`float_string`/`int_to_str`/`float_to_str` spellings removed; core slices expose allocator-first `slice.buf(a, ...)`/`slice.try_buf(a, ...)` with no default-heap wrappers and with the old public `nbuf`/`try_nbuf` spellings removed; byte helpers expose `bytes.at`, and borrowed text exposes `text.at` and allocator-first `text.dup(a, ...)`/`text.substr(a, ...)`, with the old public `byte_at` and `char_at` spellings removed; alloc has `alloc.default`, with the old `system_allocator`/`default_allocator` aliases removed; raw memory exposes namespace-bound `raw.of(seed, n)` for typed buffers, with the old public `buf_of`/`new_i32`/`new_u8` helper names removed; trace now exposes `trace.child` and `trace.set_kid` instead of public `trace_child`/`node_set_kid`; ownership modules now expose allocator-explicit `new_in`/`try_new_in` only, with default-heap `new`/`try_new` wrappers removed, and arena exports allocator-explicit `new_in`/`try_new_in` instead of public prefixed `arena_new` / `try_arena_new`. Namespace binds (`left = left`, `right = right`, `alloc = std.mem.alloc`, `arena = std.mem.arena`, `raw = std.mem.raw`, `trace = std.mem.trace`, `file = std.io.file`, `rc = std.mem.rc`, `vec = std.collections.vec`, `maps = std.collections.map`, `rt = std.concurrent.runtime`, `cown = std.concurrent.cown`, `num = std.text.num`, `slice = std.core.slice`, `text = std.text.str`, `bytes = std.text.bytes`) now prefix direct module exports so sibling/std modules can both export `thing`, `Box`, `default`, `of`, `sync`, `buf`, `integer`, `float`, `at`, or `new_in` and be used as `left.thing()` / `right.thing()` / `alloc.default()` / `raw.of(...)` / `trace.child(...)` / `file.contents(...)` / `file.out_bytes(...)` / `vec.of(...)` / `maps.of(...)` / `rt.sync(...)` / `cown.buf(...)` / `num.integer(a, ...)` / `num.float(a, ...)` / `slice.buf(a, ...)` / `text.at(...)` / `bytes.at(...)` / `arena.new_in(...)` / `rc.new_in(...)` without collision. `zenc emit <file>` now runs the same import/namespace resolver as `build` and `run`, so generated-C inspection shows `alloc__default()` instead of an alias-as-receiver artifact. Imported modules can also use their own namespace binds, so both `left = left; left.calc()` and `{ calc } = left` work when `left.zen` calls `helper.value()` through `helper = helper`.
   - Gap: destructuring imports still flatten/dedup by short name; namespace binds are source-text rewriting, not the final AST/symbol-table module system.
   - Done when: modules/types can export natural same short names without import collisions and resolver tests prove those names coexist.

2. **Generic Method Inference**
   - Status: `ReplyRef<T>.send` is generic; actor demo and a trait-impl match-arm regression prove it. Match-arm payload bindings now participate in return-type inference, so unannotated locals can infer generic receiver method returns such as `Box<str>.get()` from enum payloads in first and non-first arms. Enum match expression arms now reject the concrete C-unsafe string/non-string result mismatch before C emission.
   - Next: broaden inference coverage beyond actor replies and move match result typing toward a real common-type rule instead of first-arm result typing.

3. **Formatter**
   - Status: `zenc fmt [--check] <file>` exists; the first formatter preserves line comments, block comments, strings, char literals, and braces inside comments/literals, normalizes brace indentation/trailing whitespace, is idempotent, and has fixture tests.
   - Gap: formatter is still conservative line/brace formatting, not an AST-aware pretty-printer for the full language style.
   - Done when: `zenc fmt <file>` is idempotent, preserves comments, has fixture tests, supports check mode, and covers the full syntax/style policy.

4. **Diagnostics**
   - Status: CLI diagnostics render `error[kind]`, a human message, mapped source line/column, a source marker, and a hint for undefined names, arity errors, arg-type errors, return-fit errors, assign-fit errors, trait conformance errors, and ownership errors. The checker exposes the CLI-compatible `CheckDiagnostic { code, kind, source_offset, span_width, count, message, hint }` and first-class `Diagnostic { code, kind, span: SourceSpan, count, message, hint }` values via `diagnostic_from_source` and module diagnostic helpers, while preserving the older packed-kind oracle entrypoints; binary expressions carry source operator offsets so `main = () i32 { 1 < 2 }` reports `return-fit` at the `<` instead of falling back to a positionless diagnostic.
   - Gap: user-file line/column mapping is still assembled in the C CLI, and richer multi-diagnostic flows are not yet threaded through the validation pipeline.
   - Done when: undefined names, type errors, arity errors, and trait errors are represented as stable diagnostic values and render precise spans plus useful text.

5. **AST Import Resolution**
   - Status: resolver tests now cover namespace-bind diagnostics preserving the original user source span for unknown qualified calls and rewritten valid calls that later fail arity or argument-type checks. `std.internal.resolve` now exposes structured `ImportEdge { module, alias, namespace, start, next }` values through `import_edges`, and the checked loader consumes those edges for destructuring dependency loading and namespace-bound module loading. Import-head validation, namespace alias rewrite sets, user-module duplicate tracking, final per-name flat dedup, and namespace declaration-name collection now use source-spanned `ProvidedSymbol { name, start, next, decl_start, decl_next, imported, foreign }` values through `provided_symbols` instead of separate newline-delimited declaration scans. `ModuleTable { modules: [ModuleEntry] }` now records the transitive checked-loader module set, including namespace-bound modules and their own dependencies, and `resolve_program_data` returns `ResolvedProgram { table, flat, body_start, body_end }`; the legacy `resolve_program` wrapper returns only `flat` for the current C CLI. `resolve_parsed_program` now builds `ParsedProgram { resolved, modules, flat_decls }`, where each `ParsedModule` carries its original source, loader-stripped body, graph, and parsed declarations, so the next parse/check boundary can consume per-module AST data instead of rediscovering files from flat source. `root_link_decls` builds the root module's import library from direct graph edges, preserving namespace alias signatures and destructuring import signatures without reparsing the flat program, and `check_parsed_program` validates the root parsed module against that graph-built library. The checked loader uses the module table for import-head validation plus namespace/import closure loading before the remaining compatibility flat-source parse/check boundary. Qualified-name rewriting, protected literal/comment copying, shared source-span copying, line scanning, import/namespace line stripping, structured import-edge scanning, structured provided-symbol scanning, structured import/namespace edge loading, declaration-name collection, import-head validation, final per-name dedup, and user-declaration duplicate tracking now use the public loop-handle style instead of recursive self-calls, with large-source regressions proving namespace rewriting, long-line import stripping, protected literal/comment regions, many import/namespace lines, large import-edge lists, large provided-symbol lists, declaration-name filtering, many import-head names, final dedup over many symbols, many sibling module declarations, per-module parsing, and graph-linked root checking do not grow the call stack per token, line, symbol, or edge.
   - Gap: the C CLI still runs parse/check through `resolve_program(...).flat`; graph-linked checking currently covers the root module's direct import signatures, but full validation has not yet switched to a symbol-table/module-world resolver for every module body.
   - Done when: imports resolve through a module graph/symbol table, resolver tests pass, and positions still point to original files.

6. **JS Backend Decision**
   - Status: `compiler.genjs` is documented as an experimental computational-subset backend in [JS_BACKEND.md](JS_BACKEND.md); raw pointer/memory operations emit explicit `unsupported-in-js` markers, and tests run supported programs through Node.
   - Gap: JS is not first-class: there is no `zenc js` CLI, package/module linking story, or complete runtime/memory model.
   - Done when: docs mark JS first-class or experimental, unsupported constructs are explicit, and tests match that scope.

7. **Result/Error Policy**
   - Status: [ERROR_POLICY.md](ERROR_POLICY.md) documents the current fast/fallible naming rules and the `std.mem`, `std.core.slice`, `std.text`, `std.collections`, `std.concurrent.actor`, `std.concurrent.coroutine/sched`, `std.concurrent.cown`, and `std.io` surfaces. Tests cover allocator null lifting, arena backing allocation, namespaced ownership `try_new_in`, trace `try_tracked_in` plus `try_root_in`/`try_collect_in`, ownership/trace allocator paths, slice buffer/copy/node/concat result paths, text/string result paths, strict `try_parse_int` `Ok`/`NoDigits`/`Trailing`/`Overflow` result paths, Vec and Map fallible growth, iter map/filter result paths, actor cell/reply allocation cleanup, stateful actor `try_spawn` cleanup, coroutine spawn allocation cleanup, scheduler flag allocation failure, cown buffer/file wrapper allocation failure, and file I/O result paths.
   - Gap: the policy is still convention plus tests; effects/ownership are not checker-enforced.
   - Done when: `std.mem`, `std.text`, `std.collections`, and `std.io` document fallible vs fail-fast APIs and test both paths.

8. **Memory Safety Rules**
   - Status: [MEMORY_MODEL.md](MEMORY_MODEL.md) documents the current model; allocator-threaded tests cover arena backing storage, fallible ownership constructors, and internal AST declaration buffers; `zenc check` rejects same-body local use after `Own<T>.release_in(...)`, `Rc<T>.drop_in(...)`, or `Arc<T>.drop_in(...)` with precise ownership diagnostics.
   - Gap: no full branch-sensitive ownership, lifetime, pointer-direction, or nullability checker yet.
   - Done when: the model grows from the current local consume rule into the full pointer/ownership safety policy and tests cover representative invalid lifetime patterns.

9. **Package Manifest**
   - Status: `zenc check/build/run <project-dir>` reads `zen.toml` with `package`, `root`, `main`, optional `out`, and `ccflags` build options. A committed fixture project under `tests/fixtures/project/manifest_demo` proves `check`, `run`, and `build` against a real manifest tree.
   - Gap: no dependency graph, package registry, build profiles, or richer option model yet.
   - Done when: `zen.toml` declares package roots/options, fixture projects build from it, and package-level options/dependencies are tested.

10. **Tooling**
    - Status: `zenc fmt` exists; `zenc doc <std.mod|file.zen>` lists public declaration heads and adjacent `//` docs. Tests now assert docs for formatter, actor/concurrency, memory, collections, strict text parsing, file I/O `Result` APIs, and local source files.
    - Gap: no LSP, package tooling, or rich docs generator yet; formatter/doc output are still first-pass tools.
    - Done when: `zenc fmt` and `zenc doc` are covered by tests, and public std declarations can be listed with useful types/docs.

11. **Language Spec**
    - Status: `SPEC.md` covers current syntax, declarations, traits, generics, imports, memory, errors, concurrency, backends, tooling, and links each area to tests.
    - Gap: the spec is current-state documentation, not yet a rigorous versioned language standard with grammar and normative acceptance/rejection language.
    - Done when: spec covers syntax, declarations, traits, generics, imports, memory, errors, concurrency, and links to tests.

12. **Actor Demo Polish**
   - Status: `ActorEngine<M>` hides the raw system state, `ActorCell<M>` remains the lower-level wrapper for `tell`, `request`, `ask`, `await_reply`, and `free`, and `ActorHandle<M, ActorT>` now owns allocator-backed actor state plus its queue with receiver-scoped `tell`, `run`, `request`, `ask`, and `free` methods. `actor_demo.zen` uses namespace-bound `alloc.default()` and `actor = std.concurrent.actor`, builds a generic `ChatRoomHandle<A>` with `actor.spawn(...)`, stores the allocator pointer once in that handle, sends typed chat operations through `room.join(...)` / `room.say(...)`, and asks for `ChatStats` with `room.stats()` through `ReplyRef<ChatStats>` rather than exposing reply construction or drain/await ordering. The demo asks once after Alice joins and again after Bob joins/posts, proving actor state persists across multiple drains and that generic replies can carry a structured value instead of an actor-specific integer wrapper. Actor draining checkpoints through `std.concurrent.runtime`, keeping the raw coroutine primitive inside the runtime/coroutine substrate, and `ActorHandle.request` / `ask` now release reply storage through `ReplyRef.await` instead of duplicating raw reply-buffer reads. Setup no longer needs a type-witness message, visible raw `ActorSystem`, imported `actor_*` helpers, standalone reply construction, named request shims, wrapper refs, direct `cell` constructor imports, or allocator threading through every room operation in `main`. Actor allocation has `cell.try_reply`, `try_cell`, and stateful `actor.try_spawn` value paths for recoverable setup failure, and namespace regression tests prove `actor.cell` / `ActorCell.request` can coexist with another module exporting `cell`, while `actor.spawn` / `actor.try_spawn` can coexist with the coroutine substrate's own names.
   - Gap: broader actor spawning/scheduling ergonomics can still improve.
   - Done when: the demo remains runnable, hides queue plumbing, and no longer needs type-witness seed ceremony, manual reply allocation, or repeated allocator arguments in `main`.

Priority: generic inference, namespaces, formatter, actor demo, AST imports, Result policy, diagnostics, memory rules, manifest, tooling, spec, JS decision.
